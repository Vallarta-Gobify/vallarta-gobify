# Reports Management

The Reportes (Reports) section allows you to view, track, and manage citizen-submitted reports and requests. This module helps ensure timely responses to citizen needs and complaints.

## What Are Reports?

Reports are citizen-submitted communications that include:
- **Service requests** (e.g., assistance applications)
- **Complaints** about services or programs
- **Inquiries** about program status
- **Issues** requiring government attention
- **Feedback** on services received
- **Appeals** of decisions

## Accessing Reports

1. Click the **Reportes** tab in the main navigation
2. Reports list loads showing all submitted reports
3. List is organized by submission date (newest first)

### Report Types

Reports are classified by type:

**Solicitud (Request):**
- Requesting government assistance
- Applying for new program
- Asking for information
- Requesting appointment

**Queja (Complaint):**
- Service quality issue
- Staff behavior concern
- Program delivery problem
- Delay in processing

**Consulta (Inquiry):**
- Question about program
- Status check
- General information
- Clarification needed

**Incidencia (Incident):**
- Problem report
- Emergency situation
- Urgent issue
- Critical concern

**Sugerencia (Suggestion):**
- Improvement idea
- Feedback on service
- Process enhancement
- Program recommendation

**Otro (Other):**
- Doesn't fit other categories
- Special circumstance
- Miscellaneous

### Report Categories

Reports are further organized by subject:

- **Programa Social**: Related to social programs
- **Documentación**: Document-related issues
- **Atención Ciudadana**: Citizen service matters
- **Información**: Information requests
- **Administrativo**: Administrative issues
- **Técnico**: Technical problems with system
- **Otro**: Other categories

### Status Types

Each report has a status indicating its current state:

| Status | Color | Meaning | Action Required |
|--------|-------|---------|-----------------|
| **Pendiente** | Yellow | Awaiting review | Assign and begin review |
| **En Revisión** | Blue | Currently being reviewed | Complete assessment |
| **En Proceso** | Light Blue | Action being taken | Continue resolution |
| **Resuelto** | Green | Resolved/completed | Close or archive |
| **Cerrado** | Gray | Closed/archived | No action needed |
| **Cancelado** | Red | Cancelled | Document reason |

### Priority Levels

Reports are assigned priority based on urgency:

| Priority | Color | Response Time | Examples |
|----------|-------|---------------|----------|
| **Urgente** | Red | Same day | Emergency assistance, critical issues |
| **Alta** | Orange | 1-2 days | Benefit suspension, important appeals |
| **Media** | Yellow | 3-5 days | Standard requests, general complaints |
| **Baja** | Green | 5-10 days | Suggestions, general inquiries |

**Priority Assignment:**
- System may auto-assign based on type
- Staff can change priority
- Supervisors can override
- Urgent reports flagged prominently

### Filtering Reports

Apply filters to narrow down the list:

**By Status:**
```
1. Click "Status" filter dropdown
2. Select one or more statuses:
   - Pendiente (to see what needs attention)
   - En Revisión (to see what's being worked on)
   - Resuelto (to review completed reports)
3. List updates to show only selected statuses
```

**By Type:**
```
1. Click "Type" filter
2. Select: Solicitud, Queja, Consulta, etc.
3. View only reports of that type
```

**By Category:**
```
1. Click "Category" filter
2. Select subject area
3. See category-specific reports
```

**By Priority:**
```
1. Click "Priority" filter
2. Select: Urgente, Alta, Media, Baja
3. Focus on priority level
```

**By Date Range:**
```
1. Click "Date Range" filter
2. Select:
   - Today
   - This week
   - This month
   - Custom range
3. Enter custom dates if needed
4. Apply filter
```

**By Assigned Staff:**
```
1. Click "Assigned to" filter
2. Select staff member name
3. Or select "Unassigned" for pending assignment
4. View that person's workload
```

### Combining Filters

Use multiple filters for precise results:

**Example: Urgent Pending Complaints**
```
Status: Pendiente
Type: Queja
Priority: Urgente
Result: Shows urgent complaints needing immediate attention
```

**Example: My Assigned Reports This Week**
```
Assigned to: [Your Name]
Date Range: This week
Status: En Revisión, En Proceso
Result: Your current workload
```

**Clearing Filters:**
1. Click "Clear All Filters" button
2. Or click X on individual filter chips
3. View returns to full unfiltered list

## Managing Report Status

Update status as you process reports.

[Screenshot: Status change interface with dropdown and notes]

### Changing Report Status

```
1. Open report details
2. Click "Change Status" button
3. Select new status from dropdown:
   - Pendiente
   - En Revisión
   - En Proceso
   - Resuelto
   - Cerrado
   - Cancelado
4. Add required notes explaining change
5. Click "Update Status"
6. Status changes immediately
7. Citizen notification sent (if configured)
8. Change logged in history
```

### Status Workflow

**Typical Flow:**

```
Pendiente (submitted)
    ↓
En Revisión (staff assigned, reviewing)
    ↓
En Proceso (action being taken)
    ↓
Resuelto (issue resolved)
    ↓
Cerrado (archived, no further action)
```

**Alternative Paths:**

```
Pendiente → Cancelado (duplicate, invalid)
En Revisión → Pendiente (more info needed)
En Proceso → En Revisión (reassessing)
Resuelto → En Proceso (issue reopened)
```

### When to Use Each Status

**Pendiente (Pending):**
- Report just submitted
- Awaiting initial review
- Not yet assigned
- Requires triage

**En Revisión (Under Review):**
- Assigned to staff member
- Being investigated
- Gathering information
- Assessing situation

**En Proceso (In Progress):**
- Action plan determined
- Actively working on resolution
- Coordinating with other departments
- Citizen has been contacted

**Resuelto (Resolved):**
- Issue has been addressed
- Citizen satisfied
- Action completed
- Awaiting final confirmation

**Cerrado (Closed):**
- No further action needed
- Archived for records
- All follow-ups complete
- Final status

**Cancelado (Cancelled):**
- Duplicate report
- Invalid submission
- Citizen withdrew request
- Outside system scope

### Required Notes for Status Changes

When changing status, always add notes explaining:

**Moving to En Revisión:**
```
Example: "Assigned for review. Checking eligibility
documents and program availability."
```

**Moving to En Proceso:**
```
Example: "Documents verified. Creating program enrollment
request. Expected completion: 3 days."
```

**Moving to Resuelto:**
```
Example: "Beca aprobada. Folio BECA-2025-456. Citizen
notified via email. Enrollment effective 01/01/2026."
```

**Moving to Cancelado:**
```
Example: "Duplicate of REP-2025-001100. Citizen already
has active enrollment in this program."
```

**Notes Checklist:**
- [ ] What action was taken?
- [ ] Why was this status chosen?
- [ ] What's the next step?
- [ ] How was citizen notified?
- [ ] Any follow-up needed?

## Assigning Reports

Distribute reports to appropriate staff for handling.

### Assigning a Report

```
1. Open report details
2. Click "Assign" button
3. Select staff member from dropdown
4. Add assignment note (optional but recommended)
5. Click "Assign"
6. Staff member receives notification
7. Report appears in their workload
```

[Screenshot: Assignment dropdown with staff list]

### Assignment Best Practices

**Assign Based On:**
- Staff expertise (programs, services)
- Current workload (balance distribution)
- Language skills (if needed)
- Availability
- Priority of report

**Assignment Notes:**
Include:
- Why this person is assigned
- Special instructions
- Deadline or urgency
- Related reports

**Example:**
```
Assigned to: Juan Pérez
Note: "Juan handles scholarship programs. Priority alta -
please review by end of day. Related to folio BECA-2025-400."
```

### Reassigning Reports

If report needs different handler:

```
1. Open report
2. Click current assignee
3. Select new staff member
4. Add reassignment reason
5. Save
6. Both old and new assignee notified
```

**Reassignment Reasons:**
- Workload balancing
- Expertise match
- Staff absence
- Escalation to supervisor
- Department transfer

### Unassigning Reports

Return report to unassigned queue:

```
1. Open report
2. Click "Unassign" button
3. Add reason
4. Confirm
5. Report returns to pending pool
```

### Best Practices for Communication

**Tone:**
- Professional and courteous
- Clear and concise
- Empathetic to citizen concerns
- Action-oriented

**Content:**
- State report folio for reference
- Explain current status
- Provide timeline if possible

**Timing:**
- Acknowledge receipt within 24 hours

## Common Report Management Tasks

### Task: Process New Report

```
1. Navigate to Reportes section
2. Filter: Status = Pendiente
3. Sort: Priority (Urgent first)
4. Click first pending report
5. Review details and attachments
6. Assign to appropriate staff (or self)
7. Change status to En Revisión
8. Add initial assessment notes
9. Send acknowledgment to citizen
10. Set follow-up reminder
```

### Task: Clear Pending Queue

```
1. Filter: Status = Pendiente
2. For each report:
   a. Quick review
   b. Assign to staff
   c. Set priority
   d. Add triage notes
3. Goal: All reports assigned by end of day
4. Generate assignment summary for supervisor
```

### Task: Follow Up on In-Progress Reports

```
1. Filter: Status = En Proceso
2. Filter: Assigned to = Me
3. Review each report:
   a. Check last activity date
   b. If >3 days, update citizen
   c. Add progress notes
   d. Move to Resuelto if complete
4. Ensure no stale reports
```

### Task: Generate Weekly Report Summary

```
1. Filter: Date Range = This week
2. Filter: Status = Resuelto
3. Filter: Assigned to = Me (or Team)
4. Export to Excel
5. Analyze:
   - Total resolved: _____
   - By type
   - By category
   - Average resolution time
6. Share with supervisor
```

### Task: Handle Urgent Complaint

```
1. Receive urgent complaint notification
2. Open report immediately
3. Review citizen details and issue
4. Change priority to Urgente
5. Assign to supervisor or self
6. Change status to En Proceso
7. Contact citizen within 1 hour
8. Document all actions
9. Escalate if needed
10. Resolve same day
11. Follow up next day to confirm satisfaction
```

## Best Practices

### Daily Routine

**Morning:**
1. Check pending reports
2. Review urgent priorities
3. Respond to overnight submissions
4. Plan day's workload

**During Day:**
1. Process reports in priority order
2. Update statuses regularly

4. Document all actions

**End of Day:**

- Update status at least weekly
- Document all actions
- Maintain professional communication
- Resolve within published timeframes

**Documentation:**
- Always add notes when changing status
- Record phone conversations
- Note decisions and reasoning
- Include follow-up items

**Batch Processing:**
- Group similar reports
- Process same type together
- Use templates for common responses
- Set aside dedicated report processing time

**Prioritization:**
- Urgent first, always
- Older reports next
- Quick wins (easy resolutions)
- Complex cases when time allows

## Troubleshooting

### Cannot See Citizen Details

**Check:**
- User permissions
- Citizen record still exists
- Link is correct
- Browser issues (try refresh)

### Message Not Sending

**Verify:**
- Email address is valid
- Phone number format correct
- System email service working
- Try alternative communication method

### Report Stuck in Status

**Review:**
- Last action taken
- What's blocking progress?
- Need to reassign?
- Escalate to supervisor?
